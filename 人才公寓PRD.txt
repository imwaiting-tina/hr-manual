# 产品需求说明书：人才公寓管理模块 (V1.2)

| 文档版本 | V1.2 |
| :--- | :--- |
| **项目名称** | 人事系统 - 人才公寓管理子系统 |
| **核心变更** | **支持双人间合租场景**：针对管培生群体，支持两人合住一间双人间。补贴按房间维度发放（2000元/月/间），系统自动计算人均分摊金额。 |
| **面向对象** | 前端开发、后端开发、测试工程师、产品经理、财务/HR业务方 |

## 1. 业务背景
为吸引和保留人才，公司提供人才公寓福利。随着管培生人数的增加，出现了**双人间合租**的需求。现行规则“每人补贴2000元”若直接套用在双人间上，会导致公司成本翻倍。因此，需调整规则为：**补贴跟着“房间”走，而非跟着“人”走**。即：无论双人间住1人还是2人，该房间每月享受的补贴上限均为2000元。

## 2. 核心业务规则

### 2.1 福利判定与计算逻辑（系统自动）
系统根据员工类别和所选房型，动态计算房间的补贴总额及个人应付金额。

*   **规则A（普通员工）**：`employee_type = '正式'` 或 `'合同'`。
    *   **逻辑**：`最终租金 = 房源原价 × 0.7`
    *   **适用房型**：单间、一室一厅、双人间（按一人全额付费计算）。

*   **规则B（管培生 - 单间/多人间）**：`employee_type = '管培生'`。
    *   **逻辑**：`最终租金 = MAX(0, 房源原价 - 2000)`
    *   **说明**：单人入住单间，补贴上限2000元/月。

*   **规则C（管培生 - 双人间合租）**：`employee_type = '管培生'` 且 `room_type = '双人间'`。
    *   **逻辑**：
        1.  **房间补贴总额**：固定为 **2000元/月**（不论住几个人）。
        2.  **人均分摊**：`个人应付 = (房源原价 - 2000) / 入住人数`。
        3.  **保底原则**：若 `(原价 - 2000)` 为负数，则个人应付为 **0元**（即房租全免，且不找零）。
    *   **示例**：
        *   *场景1*：双人间原价3000元，住2人。房间补贴2000元，剩余1000元由两人分摊，每人付500元。
        *   *场景2*：双人间原价1800元，住2人。房间补贴仅需1800元（未满2000），剩余0元，两人都无需付钱。
        *   *场景3*：双人间原价3000元，住1人（管培生独居）。房间补贴2000元，该员工个人付1000元。

### 2.2 合租关系绑定
*   双人间申请需支持“发起申请”和“加入申请”两种模式。
*   系统需识别同一房间内的两名管培生，并将他们的账单关联到同一个“房间补贴池”。

### 2.3 审批流定义
`员工发起` → `部门负责人审批` → `人事负责人审批`（**重点：核对房间总价与补贴分摊逻辑**） → `最高负责人终审` → `公寓管理员安排房间` → `员工确认入住`

## 3. 角色与权限

| 角色 | 权限描述 | 重点关注 |
| :--- | :--- | :--- |
| **员工** | 发起申请、**邀请合租人**（针对双人间）、确认入住。 | 查看个人分摊金额。 |
| **公寓管理员** | 录入房源原价、分配房间号、**指定合租关系**。 | 准确录入总价，系统自动计算人均。 |
| **人事/财务** | 复核补贴逻辑。 | **确保双人间总成本补贴不超过2000元。** |

## 4. 功能模块详述

### 4.1 员工端 - 发起申请 (Create Application)
*   **表单字段**：
    *   `room_type` (下拉): **单间 / 双人间**。
    *   `roommate_info` (条件显示): **当选择“双人间”时显示**。
        *   输入项：合租人工号/姓名。
        *   逻辑：系统校验对方是否为管培生。若是，建立关联；若否，提示“仅限管培生互住双人间”。
    *   `apply_period` (日期范围): 预计入住/退房日期。
*   **逻辑**：
    *   若选择双人间且填写了合租人，系统生成一个“主申请单”，关联两个人的名字。
    *   若选择双人间但未填合租人，状态标记为“待寻合租”，允许后续补充。

### 4.2 公寓管理员端 - 房间分配（核心逻辑）
*   **操作界面**：
    *   **入住人员**：显示该房间的所有关联申请人（1人或2人）。
    *   **房源原价**：输入该房间的整月租金（管理员填写）。
    *   **系统自动计算面板**：
        *   **房间补贴总额**：固定显示 `2000.00`（若为管培生双人间）。
        *   **房间净价**：`原价 - 2000`。
        *   **人均应付金额**：`(原价 - 2000) / 入住人数`。
*   **提交逻辑**：
    *   管理员点击提交后，系统自动生成两条“入住确认”待办，分别推送给房间内的两位员工（如果有两个人）。

### 4.3 员工端 - 确认入住
*   **展示内容**：
    *   **房型**：双人间。
    *   **合租人**：张三（管培生）。
    *   **房间总价**：3000元。
    *   **房间补贴**：2000元（公司承担）。
    *   **个人应付**：500元（(3000-2000)/2）。
    *   **确认按钮**：员工仅需确认个人部分的金额。
