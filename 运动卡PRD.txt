# 产品需求说明书：HRO运动权益管理模块 (V1.1)

| 文档版本 | V1.1 |
| :--- | :--- |
| **项目名称** | 人事系统 - HRO运动权益子系统 |
| **核心模式** | **纯数字化 / 无卡化**：系统根据名单直接生成虚拟权益资产，无实体卡。 |
| **核心目标** | 实现园区运动福利的“名单即资产”。HR导入名单 -> 系统生成权益 -> 员工亮码核销。解决实体卡物流、丢失、忘带等问题。 |
| **面向对象** | 前端开发、后端开发、测试工程师、产品经理、HR业务方 |

---

## 1. 业务背景
公司获得园区运动合作名额（游泳/健身）。传统实体卡模式存在制卡成本高、下发慢、易丢失、核销难统计等问题。现改为**纯数字化权益**：
1. **HR端**：直接导入符合资格的员工名单。
2. **系统**：自动为名单中的员工生成“运动权益资产”（含唯一核销码）。
3. **员工端**：直接查看系统发放的权益，点击出示动态二维码。
4. **商家端**：扫描二维码，系统验证“该员工是否有本次活动的核销资格”，完成扣减。

## 2. 核心业务规则

### 2.1 权益生成逻辑（无卡化）
*   **触发**：HR在后台执行“发放权益”操作（单次或批量）。
*   **生成规则**：系统不再生成“卡”，而是生成**权益记录 (Entitlement)**。
    *   每一条权益记录包含：`员工ID` + `活动ID` + `唯一核销码` + `状态`。
    *   **唯一核销码**：系统后台自动随机生成（无需HR录入），用于二维码展示。
*   **数据关系**：一个员工在一次活动中只有一条权益记录。

### 2.2 核销机制（身份核验）
*   **动态二维码**：员工端展示的二维码包含 `员工ID` 和 `动态Token`。
*   **商家扫码**：
    *   系统校验该 `员工ID` 是否存在于当前活动的“已发放名单”中。
    *   校验该权益是否已被核销（防止重复使用）。
    *   校验是否在有效期内。
*   **结果**：核销成功 -> 系统标记该权益为“已使用”，并记录核销商家。

### 2.3 角色权限
| 角色 | 权限描述 | 重点关注 |
| :--- | :--- | :--- |
| **HR** | 创建活动、导入名单、作废权益、查看报表。 | 名单准确性、成本控制。 |
| **员工** | 查看名下权益、亮码、查看核销记录。 | 二维码有效性、有效期。 |
| **商家** | 扫码核销、查看本店记录。 | 核销成功率、操作简单。 |

## 3. 功能模块详述

### 3.1 HR端 - 权益活动管理
*   **创建活动**：
    *   字段：活动名称（如“2024夏季游泳福利”）、权益类型（游泳/健身）、有效期、适用商户（多选）。
*   **导入名单（核心）**：
    *   下载模板 -> 填写工号/姓名 -> 上传。
    *   系统校验：工号是否存在、是否重复发放。
    *   点击“确认发放”：系统批量生成权益记录。
*   **权益列表**：
    *   展示：工号、姓名、权益类型、状态（未使用/已使用/已过期）、核销时间。
    *   操作：作废（针对未使用的权益）、导出Excel。

### 3.2 员工端 - 我的运动权益
*   **权益列表**：
    *   展示名下所有有效的运动权益。
    *   **状态标签**：未使用、已核销、已过期。
    *   **视觉提示**：即将过期的权益标红或置顶。
*   **核销详情页**：
    *   点击权益进入详情。
    *   **核心组件**：**动态二维码**（大图展示，每分钟刷新）。
    *   **信息展示**：权益名称、适用门店、有效期、使用说明。
    *   **核销记录**：展示“何时在何地被核销”。

### 3.3 商家端 - 扫码核销（极简）
*   **登录**：账号密码登录（账号由HR创建，绑定特定商户）。
*   **主界面**：全屏“扫一扫”按钮。
*   **核销流程**：
    1.  扫描员工动态二维码。
    2.  系统瞬间验证，弹出确认框（员工姓名、权益名称、状态）。
    3.  点击确认，系统标记该员工名下的该条权益为“已使用”。
*   **记录查询**：按日/周查看本店核销记录。
