## 第一章 系统概述

### 1.1 系统定位

本系统是一款面向企业内部的人力资源管理系统，覆盖从招聘需求发起到新员工入职归档的全生命周期管理。系统以流程驱动为核心，将招聘审批、简历管理、面试安排、Offer发放、入职办理等环节整合于统一平台，实现人事业务数字化、标准化和可追溯。

### 1.2 核心业务流程

系统的核心业务流程为：**招聘需求发起 → 多级审批 → 简历筛选 → 面试安排（一面/二面/终面） → Offer发放 → 入职准备 → 入职引导 → 档案归档**

业务流程涉及以下关键节点：

| 序号 | 流程阶段 | 负责角色 | 系统对应模块 |
|------|---------|---------|------------|
| 1 | 招聘需求申请 | 用人部门 | 招聘管理 |
| 2 | 需求审批 | 部门负责人 / 人事负责人 / 最高负责人 | 招聘管理 |
| 3 | 简历筛选与入库 | 人事专员 | 招聘管理 - 简历库 |
| 4 | 面试安排与评价 | 人事 / 用人部门 / 最高负责人 | 面试安排管理 |
| 5 | 录用审批 | 最高负责人 | 面试安排管理 |
| 6 | Offer发送与确认 | 人事专员 | Offer管理 |
| 7 | 入职前准备 | 多部门协同 | 入职管理 - 任务分配 |
| 8 | 入职引导执行 | 人事 / 物业 / IT / 社保 / 商保 | 入职引导 |
| 9 | 档案归档 | 人事专员 | 员工档案 |

### 1.3 系统特点

- **流程化审批**：招聘需求需经部门负责人、人事负责人、最高负责人三级审批，确保用人规范
- **简历库管理**：集中存储从招聘平台获取的简历，支持分类检索与调取
- **多轮面试跟踪**：支持一面、二面、终面的全流程管理，面试结果在线记录
- **智能Offer管理**：支持模板化Offer邮件发送、状态追踪、自动提醒与过期处理
- **多部门协同入职**：将入职任务分解至各责任部门，逐项确认确保无遗漏
- **电子档案管理**：入职材料在线归档，支持扫描件上传与查阅

## 第二章 招聘管理

### 2.1 模块概述

招聘管理模块是系统的起点，用于管理用人部门的招聘需求从发起到获批的全过程。该模块对应纸质文件《聘用员工申请表》的线上化，涵盖需求填写、多级审批和简历入库三个核心环节。

### 2.2 业务流程

#### 2.2.1 需求发起（用人部门端）

用人部门在系统中填写招聘需求表单，字段包括：

**基础信息字段：**

| 字段名称 | 字段类型 | 填写说明 |
|---------|---------|---------|
| 申请部门 | 下拉选择 | 自动带出当前部门 |
| 招聘职位 | 文本输入 | 填写具体岗位名称 |
| 招聘人数 | 数字输入 | 该岗位计划招聘人数 |
| 建议职级 | 文本输入 | 期望的职级范围 |
| 薪酬范围 | 数字格式 | 税前月薪范围 |
| 年度预算内 | 单选 | 是 / 否 |
| 期望到岗时间 | 日期选择 | 期望新人入职日期 |

**招聘条件字段：**

| 字段名称 | 字段类型 | 可选项 |
|---------|---------|--------|
| 招聘原因 | 单选 | 业务扩大、员工离职、新设职位、其他（需补充说明） |
| 性别要求 | 多选 | 男（需填写年龄范围）、女（需填写年龄范围）、不限 |
| 学历要求 | 多选 | 博士、硕士、本科、大专、不限 |
| 工作经历 | 单选 | 一年、两年、三年、五年、十年、其他（需补充说明） |
| 户籍要求 | 单选 | 上海、非上海人、不限 |
| 优先专业 | 富文本 | 可填写多个优先专业 |
| 证书要求 | 富文本 | 岗位所需资格证书 |
| 其他要求 | 富文本 | 补充招聘条件 |
| 职位描述 | 富文本 | 岗位职责与工作内容简述 |

填写完成后，提交发布，进入审批流程。

#### 2.2.2 需求审批流程

招聘需求需经过三级审批，各级审批人操作如下：

**第一级：部门负责人审批**

- 收到招聘需求审批通知
- 查看需求详细信息
- 操作：批准（进入下一级）或 驳回（退回申请人，需填写驳回原因）

**第二级：人事负责人审批**

- 收到待审批通知
- 审核需求合理性与编制情况
- 操作：批准（进入下一级）或 驳回（退回申请人）

**第三级：最高负责人（Jenny）审批**

- 收到待审批通知
- 最终审批决策
- 操作：批准（需求生效，同步至人事端）或 驳回（退回申请人）

#### 2.2.3 简历管理（人事端）

招聘需求获批后，人事专员在系统中开展招聘工作：

1. **简历获取**：在Boss直聘等招聘平台发布岗位，收取求职者简历
2. **简历初筛**：对收到的简历进行初步筛选
3. **简历入库**：将筛选通过的简历上传至系统简历库，建立人才储备

简历库支持按职位、来源、筛选状态等维度进行检索和管理。

### 2.3 相关文件

- 《聘用员工申请表》：线上表单对应此模块的全部字段

## 第三章 面试安排管理

### 3.1 模块概述

面试安排管理模块负责管理从简历筛选通过后到录用决策前的全部面试活动。系统支持三轮面试（一面、二面、终面）的在线安排与结果记录，同时管理《求职申请表》的线下填写与扫描件归档。

### 3.2 面试流程

#### 3.2.1 第一轮面试（人事面试）

**面试安排（人事端操作）：**

1. 从简历库中调取候选人简历
2. 安排一面时间，选择面试方式：
   - 线下面试：预约会议室
   - 线上面试：添加腾讯会议的链接和会议号
3. 在该板块记录面试时间、方式等信息，便于后续流程推进

**面试结果（人事端操作）：**

面试完成后，面试官在系统中选择一面结果：推荐 / 放弃，并增设上传PDF的入口，用于上传《求职申请表》第二页第一部分的扫描件。

若结果为"推荐"，系统自动触发二面安排流程。

#### 3.2.2 第二轮面试（用人部门面试）

**面试准备：**

一面通过后，由HR线下联系候选人填写《求职申请表》第一页基础信息（纸质版），候选人填写后提交扫描件PDF存档。

**面试安排与结果（部门负责人端操作）：**

1. 收到二面通知：包括面试时间和方式，《求职申请表》的PDF
2. 进行二面
3. 选择二面的面试结果（推荐 / 放弃），并增设上传PDF的入口，用于上传《求职申请表》第二页第二部分的扫描件
4. 若推荐，填写相应求职者的拟录用信息

**拟录用信息填写（若二面推荐）：**

部门负责人填写拟录用信息，包括：

| 字段名称 | 填写说明 |
|---------|---------|
| 录用公司 | 可不填等Jenny填写 |
| 录用岗位 | 实际录用的岗位名称 |
| 直属领导 | 汇报上级 |
| 工作内容 | 岗位职责描述 |
| 建议每月税前收入 | 薪资建议 |
| 试用期税前收入 | 试用期薪资 |
| 福利 | 社会保险、商业保险、其他（多选） |
| 开始工作日期 | 预计入职日期 |
| 合同期 | 劳动合同期限 |
| 试用期 | 试用期月数 |

**入职前沟通：**

终面通过后，人事端发起入职前沟通，安排沟通时间（同面试安排机制），通知到用人部门负责人，沟通内容包括确认入职时间、薪资细节等。HR在线下与候选人进行沟通并收取相关文件。

#### 3.2.3 终面（最高负责人面试）

**面试安排：**

二面通过后，系统自动安排终面，通知最高负责人（Jenny）及黄一萧参与。

**面试结果（最高负责人端操作）：**

1. 收到终面通知：包括面试时间和方式，《求职申请表》的PDF
2. 进行终面
3. 选择终面的面试结果（推荐 / 放弃）
4. 若推荐，对部门负责人填写的拟录用信息进行审批，可做修改后确认

### 3.3 相关文件

- 《求职申请表》：采用线下填写方式，纸质版由候选人手工填写后扫描为PDF上传至系统存档。第一页为基础信息，第二页为各轮面试评价记录，第三页为建议薪资等录用信息。

## 第四章 Offer管理

### 4.1 模块概述

Offer管理模块负责录用通知书的生成、发送与跟踪。系统支持模板化Offer邮件管理、自动发送、状态追踪和智能回复识别，对应纸质文件《录用通知书》的线上化。

### 4.2 Offer发送流程

#### 4.2.1 模板管理（人事端）

1. **模板上传**：在系统中上传Offer邮件固定模板
2. **变量设置**：模板中设定以下变动字段，系统将自动链接之前录入的所有信息（简历、《求职申请表》内容）进行填充替换：
   - 候选人姓名
   - 入职公司
   - 职位
   - 职级
   - 汇报上级
   - 月工资
   - 开始工作日期
   - 试用期时长
   - 报道时间和地点

#### 4.2.2 Offer发送

系统通过公司公共邮箱自动发送Offer邮件，并提供以下功能：

**时效管理：**
- 设定回复截止时间（默认X小时），超时自动标记为"已过期"
- 临近截止未回复，自动发送提醒邮件

**状态追踪：**

| 状态 | 说明 |
|------|------|
| 待发送 | Offer已生成，待人事确认发送 |
| 已发送 | Offer邮件已发送给候选人 |
| 待回复 | 候选人已收到，等待回复 |
| 已接受 | 候选人接受Offer |
| 已拒绝 | 候选人拒绝Offer |
| 已过期 | 超过截止时间未回复 |

**智能识别：**
- 系统读取回复邮件内容，自动识别"接受"或"拒绝"意图
- 自动更新Offer状态

**过期处理：**
- 已过期Offer可选择手动延期重新发送或取消

### 4.3 录用材料收集

候选人接受Offer后，由HR在线下与候选人沟通并收取以下材料：

| 材料名称 | 用途 |
|---------|------|
| 身份证扫描件 | 身份核验 |
| 学信网学籍在线验证报告 | 学历验证 |
| 退工单扫描件（若有） | 前雇主离职证明 |
| 银行卡复印件 | 工资卡信息登记 |
| 户口本户主页和本人页复印件 | 户籍信息备案 |
| 毕业证书复印件（实习生提供学生证复印件） | 学历/学籍证明 |
| 1个月内体检报告 | 健康状况确认 |

## 第五章 入职管理

### 5.1 模块概述

入职管理模块负责管理候选人接受Offer后到正式入职前的全部准备工作。系统将入职准备任务分解到各责任部门，以任务卡形式逐项下发、逐项确认，确保入职流程完整无遗漏。

### 5.2 任务分配机制

人事端根据新员工的入职部门和岗位，自动生成分部门的入职准备任务清单，精确分配到每个具体负责人的账号内。完成一项就确认一项，人事端可见所有流程的完成情况，但只能点击确认人事部的任务，若全部完成则代表入职流程全部完成并完成员工档案的归档。

### 5.3 各部门任务清单

#### 5.3.1 人事端

| 序号 | 任务内容 | 完成标准 |
|------|---------|---------|
| 1 | 沟通跟踪动态 | 与候选人保持联系，保证按时到岗 |
| 2 | 收取相关材料并进行原件验证 | 收取身份证、学历验证报告、退工单等材料 |
| 3 | 带领员工参观公司办公场所 | 引导新员工熟悉办公环境 |
| 4 | 介绍公司同事姓名、职务 | 介绍各部门同事 |
| 5 | 加入微信群并跟进更名事宜 | 添加群聊后，发送含新人介绍的迎新公告 |
| 6 | 办理钉钉入职，填写入职登记表，设置考勤组，完成第一次打卡 | 钉钉系统完成登记 |
| 7 | 中午带领新员工午餐 | 完成入职欢迎午餐 |
| 8 | 签订劳动合同/实习协议 | 合同签署完成 |
| 9 | 签订员工手册 | 员工手册签署页签字确认 |
| 10 | 导出钉钉入职登记表用于存档 | 登记表PDF归档 |
| 11 | 员工培训 | 完成入职培训 |
| 12 | 所有文件是否完成用印和签字 | 确认文件完整性 |

#### 5.3.2 物业端

| 序号 | 任务内容 |
|------|---------|
| 1 | 安排座位和电脑 |
| 2 | 准备并发放办公用品 |
| 3 | 开设门禁卡 |

#### 5.3.3 IT端

| 序号 | 任务内容 |
|------|---------|
| 1 | 开设邮箱账户 |
| 2 | 开设本系统账户 |
| 3 | HKMS系统账户权限开设 |

#### 5.3.4 社保端

- 办理用工
- 办理社保和公积金

#### 5.3.5 商保端

- 办理商保

### 5.4 迎新公告功能

新员工确认入职后，系统自动生成迎新公告，发布至公司微信群。

**自动提取字段（来源：简历/录用表）：**

| 公告字段 | 数据来源 |
|---------|---------|
| 姓名 | 简历姓名 |
| 部门 | 录用表「分配部门」 |
| 岗位 | 录用表「录用岗位」 |
| 入职日期 | 入职确认记录 |
| 头像照片 | 简历附件或上传 |
| 自我介绍 | 简历摘要「一句话」 |
| 学历/毕业 | 简历「最高学历」 |

**生成触发逻辑：**
- 入职状态 =「已确认」
- 系统自动拉取简历字段
- 按模板生成公告文本
- HR 审核（可修改）
- 确认发布 → 写回员工档案
- 若简历无照片 → 显示系统默认头像（姓名首字母）

**公告发布方式：** 系统生成公告卡片发布

## 第六章 入职引导与档案归档

### 6.1 入职确认流程

#### 6.1.1 新员工端

新员工获得系统账号后，确认入职并对入职服务评分（满分10分）。

#### 6.1.2 用人部门端

用人部门确认员工是否到岗，并确认发出的招聘需求已完成，此为招聘闭环。

### 6.2 合同用印与档案归档

#### 6.2.1 合同用印

需盖章的合同在用印系统申请，完成盖章后上传扫描件。

#### 6.2.2 员工档案建立

上传材料包括：

| 材料类别 | 具体文件 |
|---------|---------|
| 合同类 | 劳动合同/实习协议扫描件 |
| 申请表类 | 《求职申请表》完整扫描件（含各轮面试评价） |
| 身份证明 | 身份证复印件 |
| 学历验证 | 学历验证报告 |
| 离职证明 | 退工单（如有） |
| 银行卡 | 银行卡复印件 |
| 户籍信息 | 户口本户主页和本人页复印件 |
| 毕业/学籍证明 | 毕业证书复印件（实习生提供学生证复印件） |
| 健康证明 | 1个月内体检报告 |
| 系统登记 | 钉钉入职登记表 |
| 手册确认 | 员工手册签署页 |

所有材料归档完成后，代表入职流程全部结束，系统自动建立完整的员工电子档案，此为入职闭环。

### 6.3 流程闭环机制

| 闭环节点 | 触发条件 | 系统操作 |
|---------|---------|---------|
| 招聘闭环 | 用人部门确认员工到岗 | 招聘需求状态变更为「已完成」 |
| 入职闭环 | 合同用印完成且档案材料齐全 | 员工档案建立完成，入职流程结束 |

## 第七章 系统角色与权限

### 7.1 角色定义

| 角色 | 职责描述 | 系统权限 |
|------|---------|---------|
| 用人部门 | 发起招聘需求 | 填写招聘申请、查看本部门招聘进度 |
| 部门负责人 | 审批招聘需求、参与二面 | 审批本级需求、填写二面评价、确认员工到岗 |
| 人事专员 | 执行招聘全流程 | 简历管理、面试安排、Offer发送、任务分配、档案管理 |
| 人事负责人 | 审批招聘需求 | 审批本级需求 |
| 最高负责人 | 最终审批决策、参与终面 | 终面审批、录用审批、薪资确认 |
| 物业 | 入职物资准备 | 查看分配给自己的入职任务并确认 |
| IT | 系统账户开通 | 查看分配给自己的入职任务并确认 |
| 社保专员 | 社保公积金办理 | 查看分配给自己的入职任务并确认 |
| 商保专员 | 商业保险办理 | 查看分配给自己的入职任务并确认 |
| 新员工 | 配合入职流程 | 对入职引导评分、确认入职 |

### 7.2 数据权限

- **人事端**：可查看全部招聘和入职数据，但只能点击确认人事部的任务
- **部门端**：仅可查看本部门的招聘需求和面试安排
- **任务执行端**（物业/IT/社保/商保）：仅可查看分配给自己的入职任务
- **新员工端**：仅可查看与本人相关的入职流程并进行评分
